## Architecture as built (the contract a future brief extends)

Everything from briefs 1–9 still holds. Brief 10 added **no new action, no schema change, no store change, no new pure selector** — it is theme/markup/wiring plus a tiny string normaliser. Load-bearing facts:

- **The theme source of truth is the shared suite repo (R1).** plan's plum accent, epic palette, `--dep-line`, and `#glyph-plan` now live in `/var/www/suite/shared/theme/*` and sync DOWN into `plan/public/{css,js,illos,fonts}`. **Do not edit `public/css/instrument-core.css`, `public/js/oscilloscope.js`, or `public/illos/glyphs.svg` in this repo** — they are synced artifacts; the drift gate will go red. Change them in the suite theme source, re-sync, and commit the synced copies here. `public/css/plan.css` is still plan-local and is where plan-specific styling lives (it resolves the shared tokens by name).
- **plan stays account-free and backend-free (R6).** `server.js` is still a thin static host — no new route, no auth, no session, no DB. The hub URL is a plain href in the markup, not runtime config. A grep confirms no auth/session/cookie code in plan. The `sprintplan:board` autosave envelope is untouched by any of this.
- **The return link is static, not auth-aware (R4, a deliberate logged divergence from siblings).** The siblings are auth-gated, so their return button is auth-aware and ships hidden until `/auth/whoami` succeeds. plan is free, so its link is a plain always-visible `<a>`. plan deliberately does NOT ship `suite-return.js` and does NOT copy retrospective's `return-to-suite.test.js`.
- **The title is a single-line editable `<h1>` (#10).** `singleLineTitle` normalises pasted/entered text to one line; commit-on-blur dispatches `setPlanTitle`. This is the only behavioural addition; it touches the existing title field, not the schema.

### State shape, actions, schema — UNCHANGED

Still `schemaVersion: 2`, `deps: [{ id, blockerId, blockedId }]`, **17 actions**, `validatePlan` the load-path spine. `singleLineTitle` is a new exported helper in `validate.js` but is a pure string normaliser, not a validator change.

### Pure functions / view modules

As after Brief 9, plus `singleLineTitle` in `validate.js`. No new view module from Brief 10 itself; `public/js/` is unchanged in shape (`oscilloscope.js` is now a synced foundation file, not plan-authored).

Headless browser verification: no Playwright in this repo; require it from a sibling (`/var/www/signal/node_modules/playwright`, browsers cached in `~/.cache/ms-playwright`) in a CommonJS script. The localStorage state key is `sprintplan:board` (a `{ savedAt, plan }` envelope — unwrap `.plan`); autosave fires on dispatch, not on init, and the store ALWAYS boots fresh (a valid save raises the Brief 6 resume prompt).
